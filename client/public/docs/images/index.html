üèóÔ∏è Architecture

The Advanced File Converter uses a client-server architecture with real-time communication. Here's how the system works:

System Components

  • Client (React App)
    • User Interface components built with Material UI
    • File upload and preview functionality
    • Real-time progress tracking via Socket.io
    • Conversion option controls (format selection, quality settings, etc.)
  • Server (Node.js + Express)
    • REST API endpoints for file conversions
    • File processing and conversion logic
    • Job tracking and management
    • Security and rate limiting
  • Processing Engines
    • Sharp: Image processing and conversion
    • FFmpeg: Video transcoding and manipulation
    • PDF-lib: PDF manipulation and conversion
  • Communication Layer
    • Socket.io for real-time progress updates
    • RESTful API for file uploads and downloads
  • Storage
    • Temporary file storage for uploads
    • Result storage for downloads
    • Automatic cleanup of temporary files

Conversion Workflow

  1. User selects files and conversion options in the web interface
  2. Files are uploaded to the server via a multipart form POST request
  3. Server validates files and creates conversion jobs with unique IDs
  4. Conversion process begins in the background with real-time progress tracking
  5. Client receives updates via Socket.io as the conversion progresses
  6. Once complete, converted files are available for download
  7. Temporary files are cleaned up periodically

  • Use npm run dev to run both concurrently

üìö API Endpoints

Endpoint Method Description
/api/convert/image POST Convert image files between formats
/api/convert/video POST Convert video files between formats
/api/convert/pdf POST Perform PDF operations (convert, compress, etc.)
/api/status/:jobId GET Get status of a conversion job
/api/download/:jobId GET Download a converted file
/api/download/batch GET Download multiple converted files as ZIP

Example API Usage

curl -X POST -F "file=@image.jpg" -F "format=png" -F "quality=90" http://localhost:5000/api/convert/image
curl -X POST -F "file=@video.mp4" -F "format=webm" -F "resolution=720p" http://localhost:5000/api/convert/video
curl -X POST -F "files[]=@file1.pdf" -F "files[]=@file2.pdf" http://localhost:5000/api/merge/pdf

üîç Configuration

The application can be configured using environment variables or a .env file in the root directory.

# Server configuration
PORT=5000
NODE_ENV=development

# File storage
UPLOAD_DIR=./uploads
OUTPUT_DIR=./output
MAX_FILE_SIZE=100mb

# Processing options
MAX_CONCURRENT_JOBS=3
JOB_TIMEOUT=600000
CLEANUP_INTERVAL=3600000
