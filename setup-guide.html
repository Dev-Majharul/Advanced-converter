Advanced File Converter

Setup and Deployment Guide

Overview

This is a comprehensive guide to help you set up and deploy the Advanced File Converter application. This page appears when you first upload the project files and will be replaced by the actual application once it's properly set up.

Note: This setup guide is only visible before the application is built and deployed. After successful deployment, users will see the actual application interface.

Prerequisites

Before setting up the Advanced File Converter, ensure you have the following installed:

  • Node.js (version 18 or higher)
  • npm (version 9 or higher)
  • Git (optional, for version control)
Checking your Node.js version: 
node -v
Checking your npm version: 
npm -v

If you need to install or update Node.js, visit nodejs.org.

Installation Options

Setting Up for Local Development

Clone or unzip the project

If using Git:

git clone <repository-url>
cd advanced-converter

If you've downloaded a ZIP file, extract it and navigate to the extracted folder.

Install dependencies

npm run install-all

This command installs dependencies for both the server and client applications.

Start the development servers

npm run dev

This command starts both the backend server and the React development server concurrently.

The application will be available at http://localhost:3000

Deploying to Your Own Server

Upload project files to your server

Use SFTP, SCP, or other file transfer method to upload the project files to your server.

Install dependencies

npm run install-all

Build the client application

npm run build

This creates optimized files in the client/build directory.

Set environment variables

# Required
export NODE_ENV=production

# Optional
export PORT=80  # or your preferred port

Start the server

npm start

For persistent operation, use a process manager like PM2:

npm install -g pm2
pm2 start npm -- start
Note for Apache/Nginx: If you're using Apache or Nginx as a reverse proxy, make sure to configure it to forward requests to your Node.js application.

Deploying to Netlify

Push your code to GitHub

Netlify integrates best with code hosted on GitHub, GitLab, or Bitbucket.

Sign up/login to Netlify

Go to netlify.com and create an account or log in.

Create a new site

Click "New site from Git" and select your GitHub repository.

Configure build settings

  • Build command: npm run netlify-build
  • Publish directory: client/build

Netlify will automatically detect the netlify.toml file, but you can override the settings here if needed.

Set environment variables

Go to Site settings → Build & deploy → Environment:

  • NODE_ENV: production

Deploy your site

Click "Deploy site" and Netlify will build and deploy your application.

Important: Netlify has limitations for serverless functions. Complex operations like video conversion might require additional setup or third-party services.

Deploying to Vercel

Push your code to GitHub

Vercel integrates seamlessly with GitHub, GitLab, and Bitbucket.

Sign up/login to Vercel

Go to vercel.com and create an account or log in.

Import your project

Click "Import Project" and select your GitHub repository.

Configure deployment

Vercel will automatically detect the vercel.json configuration file, so no additional settings are needed.

Set environment variables

Go to Settings → Environment Variables to add any needed variables:

  • NODE_ENV: production

Deploy your site

Click "Deploy" and Vercel will build and deploy your application.

Important: Vercel has execution limits for serverless functions. Heavy operations like large file conversions might require adjustments or external services.

Troubleshooting

Common Issues

Node.js version issues:

If you encounter errors related to ES modules or syntax, ensure you're using Node.js version 18 or higher.

node -v
Port conflicts:

If the server cannot start due to port conflicts, you can change the port in your environment variables:

export PORT=3001  # or any available port
Build failures:

If the build process fails, check if all dependencies are installed:

npm run install-all

Deployment Check

You can run our deployment check script to verify your setup:

npm run deploy-check

This will validate that all necessary files and configurations are in place.
